# 开发 Agent 入门（分享版草稿）

## 1. Agent 到底是什么

一句话定义：Agent 是一个能够围绕目标，自主感知信息、做决策、调用工具并持续执行的系统。

和普通聊天机器人的区别在于：

- 聊天机器人偏“回答问题”
- Agent 偏“完成任务”

你给它一个目标，比如“帮我做一份上个月销售复盘”，它会自己拆解步骤、查数据、调用工具、整理结论，而不是只给你一句泛泛回答。

---

## 2. Agent 的核心组成

一个可落地的 Agent，通常由四部分构成：

1. 大脑（模型）
   - 负责理解任务、规划步骤、判断下一步动作
2. 工具（Tool Use）
   - 负责执行外部动作，如查数据库、调 API、发消息、写文件
3. 记忆（Memory）
   - 短期记忆：当前会话上下文
   - 长期记忆：历史偏好、业务知识、案例经验
4. 执行循环（Loop）
   - 典型循环是：感知 → 规划 → 行动 → 观察 → 再规划，直到完成目标

---

## 3. Agent 的工作流（以“分析销售数据”为例）

用户输入：分析上个月销售数据，并给出异常原因和优化建议。

Agent 内部过程：

1. 理解任务目标和输出格式
2. 判断需要调用哪些工具
3. 调用数据工具拉取原始数据
4. 基于结果继续补充查询
5. 汇总并生成最终报告

这个循环会反复执行，直到目标完成或达到停止条件。

### 4.3 Controller：返回 SSE 流

```js
const { PassThrough } = require("stream");

module.exports = {
  ask: async (ctx) => {
    ctx.set({
      Connection: "keep-alive",
      "Cache-Control": "no-cache",
      "Content-Type": "text/event-stream; charset=utf-8",
    });

    const stream = new PassThrough();
    ctx.status = 200;
    ctx.body = stream;

    const params = ctx.request.body || {};
    await ctx.service.aiAgent.sendMessage(ctx, { params, stream });
  },
};
```

---

### 4.4 Service：参数校验 + 调用 MCP Client

```js
module.exports = {
  sendMessage: async (ctx, { params, stream }) => {
    try {
      const question = (params?.question || "").trim();
      if (!question) {
        stream.write(`data: ${encodeURIComponent("请提供您的问题")}\n\n`);
        stream.write("data: [DONE]\n\n");
        stream.end();
        return;
      }

      await ctx.mcp.processQuery({ question, stream });
    } catch (error) {
      stream.write(`data: ${encodeURIComponent("服务异常，请稍后重试")}\n\n`);
      stream.write("data: [DONE]\n\n");
      stream.end();
    }
  },
};
```

### 4.5 MCP Client：连接模型 + 调用工具

```js
const OpenAI = require("openai");
const path = require("path");
const { Client } = require("@modelcontextprotocol/sdk/client/index.js");
const { StdioClientTransport } = require("@modelcontextprotocol/sdk/client/stdio.js");

class McpClient {
  constructor() {
    this.client = new OpenAI({
      apiKey: process.env.LLM_API_KEY,
      baseURL: process.env.LLM_BASE_URL,
    });

    this.mcp = new Client({
      name: "demo-mcp-client",
      version: "1.0.0",
    });

    this.tools = [];
  }

  async connectToServer() {
    const serverScript = path.resolve(__dirname, "../mcpServer/mcpSseServer.js");
    const transport = new StdioClientTransport({
      command: "node",
      args: [serverScript],
    });

    await this.mcp.connect(transport);
    const toolsResult = await this.mcp.listTools();
    this.tools = toolsResult.tools.map((tool) => ({
      type: "function",
      function: {
        name: tool.name,
        description: tool.description,
        parameters: tool.inputSchema,
      },
    }));
  }
}

module.exports = { McpClient };
```

### 4.6 MCP Server：暴露工具

```js
const { z } = require("zod");
const { McpServer } = require("@modelcontextprotocol/sdk/server/mcp.js");
const { StdioServerTransport } = require("@modelcontextprotocol/sdk/server/stdio.js");
const { getClicktagInfo } = require("./mcpAdapter.js");

const mcpServer = new McpServer({
  name: "demo-mcp-server",
  version: "1.0.0",
});

mcpServer.tool(
  "get_clicktag_info",
  "按埋点 key 查询信息",
  {
    clicktags: z.string().min(1),
  },
  async ({ clicktags }) => {
    const data = await getClicktagInfo({ clicktags });
    return {
      content: [{ type: "text", text: JSON.stringify(data) }],
    };
  }
);

async function main() {
  const transport = new StdioServerTransport();
  await mcpServer.connect(transport);
}

main();
```

### 4.7 前端消费 SSE（POST）

```js
import { fetchEventSource } from "@microsoft/fetch-event-source";

const controller = new AbortController();
let currentMessage = "";

await fetchEventSource("/api/aiAgent/ask", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ question: inputValue.trim() }),
  signal: controller.signal,
  openWhenHidden: true,
  onmessage(event) {
    if (event.data === "[DONE]") {
      setIsLoading(false);
      return;
    }
    currentMessage += decodeURIComponent(event.data);
    renderMessage(currentMessage);
  },
  onclose() {
    setIsLoading(false);
  },
  onerror() {
    setIsLoading(false);
    throw new Error("流式请求失败");
  },
});
```

## 5. 这份草稿里原本容易踩坑的点（已修正）

1. 密钥泄漏风险  
   原草稿里有 `apiKey: "xxx"` 这类写法。分享文档里必须强调：只用环境变量，不在代码和截图里暴露密钥。

2. MCP 工具参数映射不完整  
   工具定义要传完整 JSON Schema，不建议只传 `properties`，否则会丢失 `type` 和 `required`。

3. SSE 结束信号不统一  
   前后端都应约定 `[DONE]`，并保证任何异常分支也会写入结束标记并 `end()`。

4. 输入校验不足  
   `question` 需要 `trim`、空值判断、长度限制，避免无效请求和滥用。

5. 生产能力缺口  
   至少补上限流、鉴权、超时、重试、日志脱敏与工具调用审计。

## 6. 进阶建议（从“能跑”到“能用”）

- 增加会话记忆，支持多轮上下文
- 给工具调用增加白名单和权限隔离
- 为不同任务设计不同系统提示词模板
- 增加观测面板，记录 token、延迟、工具成功率
- 做失败降级，工具异常时给出可解释答复

---

## 7. 一页总结

Agent 不是“更会聊天”的机器人，而是“能闭环执行”的系统。  
入门最短路径是：模型 + 工具 + 循环 + 流式交互。  
如果用于真实业务，安全与稳定性必须与功能一起设计。
